# ARCHI Agent Studio — Claude Code 단일 마스터 프롬프트

당신은 지금부터 Claude Code로서 이 프로젝트의 수석 제품 아키텍트, 프론트엔드 리드 엔지니어, 백엔드 리드 엔지니어, AI 에이전트 시스템 엔지니어, UX 디렉터, 기술 PM 역할을 동시에 수행한다.

이 지시문은 하나의 단일 마스터 지시문이다.

이 프로젝트를 절대로 다음처럼 다루지 마라.

- 빠르게 만든 MVP
- 바이브코딩으로 대충 만든 프로토타입
- 보여주기용 데모
- 단순 AI 채팅 앱
- 정적인 폼 에디터
- 프롬프트만 감싼 얕은 AI 래퍼
- 가짜 기능이 들어간 목업 서비스

이 프로젝트는 건축·인테리어 전문가를 위한 상용급 B2B SaaS 플랫폼이다.

제품명은 다음과 같다.

ARCHI Agent Studio

이 플랫폼의 핵심 목적은 건축·인테리어 전문가가 AI 에이전트와 함께 다음과 같은 산출물을 만들 수 있게 하는 것이다.

- 전문 기술문서
- 시공기준서
- 법규·기준 해설서
- 블로그 포스팅
- 고객 제안서
- 이미지가 포함된 문서
- 표
- 그래프
- 계산식
- 도표
- 시공 상세도
- 자재 비교표
- 하자 분석 보고서
- 공정표
- 견적 설명 문서
- 온톨로지 기반 지식 그래프
- 내부 교육자료
- 협회 연구자료 스타일 문서

현재 화면 구조는 잘못되어 있다.

현재 UI는 사실상 다음과 같이 동작한다.

- 좌측: 단순 채팅 패널
- 우측: 정적인 폼 기반 문서 입력 화면

이 구조는 적합하지 않다.

목표 제품은 다음처럼 동작해야 한다.

- 좌측: AI 명령 센터
- 중앙: 지능형 블록 문서 캔버스
- 우측: 문서 구조 / 블록 속성 / 출처 / 지식 / 내보내기 패널
- 배경 시스템: 온톨로지 메모리 엔진과 AI 액션 프로토콜

최종 제품은 다음 도구들의 장점을 결합한 전문 AI 워크스페이스처럼 느껴져야 한다.

- ChatGPT 수준의 지능형 대화
- Notion 스타일의 블록 에디터
- Miro / FigJam 스타일의 캔버스 인터랙션
- Obsidian 스타일의 지식 그래프
- Cursor 스타일의 AI 명령 실행
- 전문 문서 출판 시스템
- 건축·인테리어 기술 지식베이스
- AI 에이전트 업무 실행 시스템

가짜 기능을 만들지 마라.
하드코딩된 데모 데이터를 실제 기능처럼 보이게 하지 마라.
조용히 실패하지 마라.
모든 내용을 단순 텍스트 하나로 납작하게 만들지 마라.
정적인 입력폼으로 구현하지 마라.
거칠고 성의 없는 AI 생성 MVP처럼 보이게 만들지 마라.

코드를 수정하기 전에 반드시 기존 코드베이스를 먼저 분석하라.
그 다음 계획을 세우고, 단계별로 구현하라.

============================================================
0. 절대 제품 원칙
============================================================

문서가 제품의 중심이다.

채팅은 최종 결과물이 아니다.
채팅은 명령 센터다.

우측 에디터는 수동적인 폼이 아니다.
우측 에디터는 지능형 블록 캔버스다.

모든 블록은 독립적인 1급 객체다.

모든 블록은 선택될 수 있어야 한다.
선택된 블록은 AI의 작업 대상이 되어야 한다.
AI는 선택된 블록에 글을 쓰고, 다시 쓰고, 교체하고, 추가하고, 생성하고, 삽입하고, 보강하고, 변환하고, 내보내고, 지식화할 수 있어야 한다.

핵심 구조는 다음과 같다.

프로젝트는 문서를 포함한다.
문서는 지능형 블록들의 조합으로 구성된다.
블록은 선택될 수 있다.
AI 에이전트는 구조화된 액션을 통해 선택된 블록을 편집한다.
블록은 텍스트, 이미지, 표, 차트, 계산식, 도표, 법규, 시공 디테일, Q&A, 출처, 참고자료를 포함할 수 있다.
생성된 블록은 온톨로지 지식으로 전환될 수 있다.
누적된 온톨로지는 에이전트를 점점 더 똑똑하게 만든다.
완성된 문서는 PDF, DOCX, Markdown, TXT, HTML, JSON 등으로 내보내거나 블로그 포스팅, 기술문서, 제안서, 지식자산으로 변환될 수 있다.

이것은 바이브코딩 MVP가 아니다.
이것은 건축·인테리어 전문 AI 문서 운영체제다.

1. 제품 비전
============================================================

ARCHI Agent Studio는 건축·인테리어 지식작업을 위한 AI 운영체제다.

사용자는 다음을 할 수 있어야 한다.

1. 워크스페이스 생성
2. 프로젝트 생성
3. 프로젝트 안에 하나 이상의 문서 생성
4. 여러 개의 지능형 블록으로 문서 구성
5. 문서 캔버스에서 아무 블록이나 선택
6. 좌측 AI 명령 패널에서 AI 에이전트와 대화
7. AI가 선택된 블록 안에 직접 글 작성, 재작성, 생성, 삽입, 교체, 분석, 보강
8. 텍스트, 이미지, 표, 차트, 계산식, 도표, 법규 근거, 시공기준, 자재 스펙, 리스크 경고, 체크리스트, Q&A, SEO 메타데이터, 블로그 콘텐츠 생성
9. 여러 블록을 컨테이너 블록으로 그룹화
10. AI에게 컨테이너 전체 완성을 지시
11. 완성 문서를 PDF, DOCX, Markdown, TXT, HTML, JSON으로 내보내기
12. 생성된 문서 내용을 온톨로지 지식 노드로 변환
13. 누적된 지식을 Obsidian 스타일 방사형 그래프로 보기
14. 누적 지식을 바탕으로 다음 AI 답변의 정확도 향상
15. 동일 시스템으로 다음 자료 생성
    - 네이버 블로그 포스팅
    - 인테리어 기술자료
    - 시공기준서
    - 법규해설서
    - 고객 제안서
    - 자재 비교표
    - 하자 분석 보고서
    - 공정표
    - 견적 설명 문서
    - 내부 교육자료
    - 협회 연구자료 스타일 문서

2. 현재 UI 문제점
============================================================

현재 화면은 다음 문제가 있다.

- 좌측 AI 패널과 우측 에디터가 서로 분리되어 보인다.
- 우측 문서 영역이 살아있는 캔버스가 아니라 정적인 입력폼처럼 보인다.
- 블록이 시각적으로는 존재하지만 능동적인 지능형 객체처럼 느껴지지 않는다.
- 사용자는 AI가 현재 어느 블록을 편집할지 명확히 알기 어렵다.
- 채팅 결과가 문서로 변환되지 않고 채팅 안에만 머무르는 느낌이 강하다.
- 우측 블록 목록은 실제 문서 구조 컨트롤러가 아니라 단순 목차처럼 보인다.
- 다음 상태들이 명확히 구분되지 않는다.
  - 프로젝트
  - 문서
  - 블록
  - 선택된 블록
  - 컨테이너 블록
  - AI 생성 블록
  - 사용자 편집 블록
  - 지식 추출된 블록
  - 내보내기 준비 완료 블록
- 전체 제품이 전문 플랫폼이 아니라 초기 AI 데모처럼 보인다.

개선된 인터페이스에서는 사용자가 항상 다음을 명확히 이해해야 한다.

- 어떤 프로젝트가 열려 있는가
- 어떤 문서가 열려 있는가
- 어떤 블록이 선택되어 있는가
- AI가 무엇을 편집할 예정인가
- AI 결과가 채팅에만 표시되는가, 문서에 적용되는가
- 생성 결과가 텍스트, 이미지, 표, 차트, 계산식, 도표, 출처, 법규 블록 중 무엇인가
- 해당 블록이 지식 그래프에 저장되었는가
- 문서가 내보내기 가능한 상태인가

============================================================
3. 목표 레이아웃
============================================================

애플리케이션은 4개 영역으로 구성된 전문 워크스페이스로 재설계해야 한다.

------------------------------------------------------------
영역 1: 글로벌 네비게이션 사이드바
------------------------------------------------------------

가장 왼쪽 사이드바다.

어둡고, 컴팩트하며, 전문적인 느낌이어야 한다.

메뉴 항목:

- 홈
- 프로젝트
- 문서
- AI 에이전트
- 지식베이스
- 온톨로지
- 메신저
- 작업센터
- 템플릿
- 내보내기
- 설정

요구사항:

- 명확한 아이콘 사용
- 간결한 간격 유지
- 현재 활성 메뉴 하이라이트
- 불필요하게 넓은 폭 사용 금지
- 엔터프라이즈 SaaS 네비게이션처럼 보여야 함

------------------------------------------------------------
영역 2: AI 명령 패널
------------------------------------------------------------

왼쪽에서 두 번째 패널이다.

이 영역은 단순 챗봇이 아니다.
AI 명령 센터다.

포함 요소:

- AI 에이전트 선택
- 모델 / 제공자 선택
- 작업 모드 선택
- 현재 작업 대상 표시
- 선택된 블록 컨텍스트 칩
- 대화 기록
- AI 액션 미리보기
- 적용 / 거절 / 다시 생성 컨트롤
- 프롬프트 입력창
- 빠른 명령 칩

AI 작업 모드:

- 글 작성
- 기술문서
- 블로그 포스팅
- 법규·기준
- 시공 디테일
- 이미지 생성
- 표·그래프
- 지식 추출
- 제안서
- Q&A

AI 패널은 현재 작업 대상을 명확히 보여줘야 한다.

예시:

현재 작업 대상
프로젝트: 단열 기준 자료
문서: 인테리어 단열시공 기준
블록: 계산식 블록 · 열관류율 계산
모드: 기술문서

프롬프트 입력창의 placeholder는 선택된 블록 타입에 따라 바뀌어야 한다.

------------------------------------------------------------
영역 3: 메인 문서 캔버스
------------------------------------------------------------

이 영역이 제품의 핵심이다.

문서는 전문적인 편집 가능한 페이지 / 캔버스로 보여야 한다.

캔버스는 다음을 지원해야 한다.

- 순서가 있는 블록
- 중첩 블록
- 컨테이너 블록
- 드래그 앤 드롭 순서 변경
- 블록 복제
- 블록 삭제
- 블록 타입 변환
- 위 / 아래 블록 삽입
- 선택 블록에 AI 생성
- 선택 블록 AI 재작성
- 선택 블록 AI 확장
- 선택 블록 AI 요약
- AI 자식 블록 생성
- 선택 블록 지식 추출

블록 클릭 시:

- 선택된 블록에 명확한 active outline 표시
- 블록 툴바 표시
- selectedBlockId 업데이트
- AI 패널의 선택 블록 컨텍스트 업데이트
- 우측 인스펙터의 블록 속성 업데이트
- 문서 구조 패널의 해당 블록 하이라이트

문서 캔버스는 정적인 폼처럼 보이면 안 된다.

살아있는 지식 문서를 블록으로 조립하는 느낌이어야 한다.

------------------------------------------------------------
영역 4: 우측 인스펙터 패널
------------------------------------------------------------

우측 패널은 탭 구조로 구성한다.

탭:

1. 문서 구조
2. 블록 속성
3. 출처
4. 지식
5. 내보내기

문서 구조 탭:

문서의 블록 트리를 보여준다.

각 항목은 다음을 표시한다.

- 블록 순서
- 블록 타입
- 블록 제목
- 콘텐츠 상태
- AI 생성 배지
- 지식 추출 배지
- 경고 / 오류 상태

블록 속성 탭:

선택된 블록의 상세 정보를 보여준다.

- 블록 ID
- 블록 타입
- 제목
- 콘텐츠 상태
- 출처 참조
- 생성 프롬프트
- 마지막 편집자
- 생성일
- 수정일
- 내보내기 포함 여부
- 온톨로지 상태

출처 탭:

다음을 보여준다.

- 법규 / 기준 참조
- 업로드된 원본 문서
- URL
- 인용 정보
- 출처 신뢰도
- 출처 상태

지식 탭:

선택된 블록에서 추출된 엔티티와 관계를 보여준다.

내보내기 탭:

다음을 보여준다.

- 내보내기 형식
- 문서 완성도 점수
- 누락된 필수 블록
- 경고
- 내보내기 기록
- 다운로드 버튼

============================================================
4. 블록 우선 아키텍처
============================================================

문서는 하나의 긴 HTML 문자열이 아니다.

문서는 순서가 있는 블록 트리다.

각 블록은 독립적인 1급 엔티티다.

최소 Block 모델:

Block {
  id: string
  workspaceId: string
  projectId: string
  documentId: string
  parentBlockId?: string
  type: BlockType
  title?: string
  content: BlockContent
  order: number
  depth: number
  status: "empty" | "draft" | "generated" | "edited" | "approved" | "error"
  aiGenerated: boolean
  locked: boolean
  exportable: boolean
  ontologyStatus: "not_extracted" | "pending_review" | "approved" | "rejected"
  sourceRefs: SourceRef[]
  metadata: Record<string, any>
  createdAt: string
  updatedAt: string
}

필수 블록 타입:

1. document_meta
2. heading
3. paragraph
4. rich_text
5. image
6. image_gallery
7. before_after
8. table
9. chart
10. formula
11. diagram
12. code
13. callout
14. quote
15. checklist
16. qna
17. law_reference
18. construction_standard
19. construction_detail
20. material_spec
21. cost_table
22. schedule
23. risk_warning
24. source_reference
25. seo_meta
26. cta
27. container
28. blog_section
29. technical_section

5. 컨테이너 블록
============================================================

컨테이너 블록은 필수 기능이다.

컨테이너 블록은 자식 블록들을 포함한다.

예시:

컨테이너:
단열두께 설계 기준

자식 블록:
- 개요 paragraph
- 법규 기준 law_reference
- 열관류율 계산 formula
- 지역별 열관류율표 table
- TDR 설명 formula
- 결로방지 상세도 image
- 1미터의 법칙 technical_section
- 시공 체크리스트 checklist
- 현장 Q&A qna
- 출처 source_reference

사용자가 컨테이너를 선택하고 다음처럼 말할 수 있어야 한다.

"이 컨테이너를 전문 기술자료 수준으로 완성해줘."

AI는 자식 블록들을 채우거나 업데이트하는 액션 플랜을 만들어야 한다.

컨테이너 동작:

- 컨테이너 선택 시 자식 블록도 약하게 하이라이트
- AI가 컨테이너 전체에 대해 작업 가능
- 내보내기 시 계층 구조 유지
- 온톨로지 추출을 컨테이너 전체에 대해 실행 가능
- 접기 / 펼치기 지원
- 재사용 가능한 템플릿으로 변환 가능
- 복제 가능
- 그룹 단위 이동 가능

6. AI 액션 프로토콜
============================================================

AI 출력은 단순 텍스트로만 처리하면 안 된다.

AI는 구조화된 액션을 반환해야 한다.

AI Action Protocol을 구현하라.

AIActionResponse {
  message: string
  actions: AIAction[]
  requiresUserApproval: boolean
  target: {
    workspaceId: string
    projectId: string
    documentId: string
    blockId?: string
    scope: "selected_block" | "selected_container" | "document" | "project"
  }
}

AIAction {
  id: string
  type: AIActionType
  targetBlockId?: string
  payload: Record<string, any>
  confidence?: number
  sourceRefs?: SourceRef[]
}

AIActionType:

- replace_block_content
- append_to_block
- insert_block_after
- insert_block_before
- create_child_block
- create_container
- update_block_title
- convert_block_type
- generate_image
- edit_image_inpaint
- create_table
- update_table
- create_chart
- update_chart
- create_formula
- update_formula
- create_diagram
- attach_source
- extract_ontology
- create_knowledge_nodes
- create_knowledge_edges
- mark_block_approved
- export_document
- create_blog_post_structure
- create_technical_document_structure

파괴적 작업이나 여러 블록을 수정하는 작업은 적용 전에 반드시 미리보기를 보여줘야 한다.

예시 AI 액션 미리보기:

AI가 다음 작업을 제안합니다.

- "법규 기준" 블록 채우기
- "열관류율 계산식" 블록 생성
- "단열재별 비교표" 블록 삽입
- "결로방지 상세도" 이미지 블록 생성
- "출처" 블록 연결

버튼:

[전체 적용]
[선택 적용]
[거절]
[다시 생성]

7. 선택 블록 컨텍스트

- 수동 노드 생성
- 수동 관계 생성

지식 추출 흐름:

1. 사용자가 블록을 생성하거나 편집
2. 시스템이 해당 블록을 지식 추출 가능 상태로 표시
3. 사용자가 "지식 추출"을 클릭하거나 AI가 추출 제안
4. AI가 노드와 관계 추출
5. 추출된 지식은 pending_review 상태로 들어감
6. 사용자가 승인하거나 수정
7. 승인된 지식은 이후 AI 답변에 사용됨

최소 테스트:

1. 블록 선택 시 AI 패널 컨텍스트 업데이트
2. AI replace_block_content 액션이 선택 블록을 업데이트
3. AI create_table 액션이 표 블록 생성
4. AI create_chart 액션이 차트 블록 생성
5. 컨테이너 블록이 자식 블록을 포함
6. 문서 구조 패널이 블록 계층을 반영
7. export serializer가 텍스트, 이미지, 표, 차트, 계산식 처리
8. 지식 추출이 pending 노드와 관계 저장
9. API 키 누락 시 가짜 출력 대신 오류 표시
10. 블록 순서 변경 시 order 보존

E2E 시나리오:

시나리오 1:
기술문서 생성 -> 문단 블록 선택 -> AI에게 작성 요청 -> 콘텐츠가 블록 안에 표시

시나리오 2:
표 블록 선택 -> AI에게 자재 비교표 요청 -> 표가 블록 안에 표시

시나리오 3:
이미지 블록 선택 -> AI에게 이미지 생성 요청 -> 이미지 블록에 생성 상태와 결과 또는 명확한 오류 표시

시나리오 4:
시공 사진으로 블로그 포스트 생성 -> AI가 블록 구조 생성 -> Markdown으로 내보내기

시나리오 5:
문서에서 온톨로지 추출 -> 노드가 지식 검수함에 표시 -> 승인 -> 그래프 뷰에 노드 표시

32. 수용 기준
============================================================

다음이 충족되어야 작업을 수용할 수 있다.

1. 블록 클릭 시 명확히 선택됨
2. AI 패널이 선택된 블록을 명확히 표시
3. 블록 선택 상태에서 프롬프트를 보내면 해당 블록이 업데이트될 수 있음
4. AI 응답은 단순 텍스트가 아니라 구조화된 액션을 생성할 수 있음
5. 사용자가 AI 액션을 미리보고 적용할 수 있음
6. 문서는 구조화된 블록으로 구성됨
7. 우측 문서 구조가 실제 블록 계층을 반영
8. 컨테이너 블록이 자식 블록을 포함 가능
9. 텍스트, 이미지, 표, 차트, 계산식, 법규/출처, 체크리스트, Q&A 블록이 각각 별도 렌더러로 존재
10. 내보내기 파이프라인이 존재하고 블록 구조를 보존
11. 지식 추출 워크플로우가 pending review를 포함한 실제 데이터 흐름으로 존재
12. API 키 누락 또는 provider 사용 불가 시 명확한 오류 표시
13. UI가 거친 AI 데모가 아니라 전문 SaaS처럼 보임
14. 완료된 기능처럼 보이기 위한 가짜 생성 콘텐츠 하드코딩 없음
15. 코드가 유지보수 가능하고 확장 가능함
